# コードスタイル整理計画

## 概要

プロジェクト全体の C/C++ ソースコード（約940ファイル）を対象に、
旧式のコーディングスタイルをモダンな Visual Studio 準拠に統一する。

## 対象外ファイル

サードパーティライブラリは変更しない:
- `Lib/third_party/` (lodepng.cpp, lodepng.h)
- `Lib/Audiere/` (Audiere.h)
- `Lib/Png/` (png.h, pngconf.h)

また、以下のフォルダはツール管理領域のため対象外:
- `.claude/`

## 整理項目

### 項目1: ファイルヘッダをDoxygen形式に変換

作成日付・著作権・内容の情報は保持し、Visual Studio / Doxygen 準拠の `///` 形式に変換する。

**変更前:**
```cpp
/* Copyright(C)URARA-works 2005 */
/* ========================================================================= */
/* ファイル名：	Img32.cpp													 */
/* 内容：		32bitイメージクラス 実装ファイル							 */
/* 作成：		年がら年中春うらら(URARA-works)								 */
/* 作成開始日：	2005/01/24													 */
/* ========================================================================= */
```

**変更後:**
```cpp
/// @file Img32.cpp
/// @brief 32bitイメージクラス 実装ファイル
/// @author 年がら年中春うらら(URARA-works)
/// @date 2005/01/24
/// @copyright Copyright(C)URARA-works 2005
```

- 「説明」フィールドがある場合は `@details` として追加
- 「作成」行がないファイル（Phase 2以降で追加したもの等）はそのまま省略

### 項目2: 関数ヘッダコメントの削除

関数ごとの定型コメントブロックを削除する。関数ヘッダと関数本体の間の空行も削除。

**変更前:**
```cpp
/* ========================================================================= */
/* 関数名：	CImg32::CImg32													 */
/* 内容：	コンストラクタ													 */
/* 日付：	2005/01/24														 */
/* ========================================================================= */

CImg32::CImg32(void)
{
```

**変更後:**
```cpp
CImg32::CImg32(void)
{
```

- 関数名・日付は git 履歴で追跡可能
- 内容説明が非自明な場合のみ `/// @brief` 1行コメントに置換

### 項目3: 関数呼び出し時の空白を削除

関数名と `(` の間の空白を削除する。

**変更前:**
```cpp
InitData ();
ZeroMemory (&m_bmpInfo, sizeof (m_bmpInfo));
SAFE_DELETE (m_pDibBack);
```

**変更後:**
```cpp
InitData();
ZeroMemory(&m_bmpInfo, sizeof(m_bmpInfo));
SAFE_DELETE(m_pDibBack);
```

- `sizeof (x)` → `sizeof(x)` も対象
- `if (`, `for (`, `while (`, `switch (`, `return (` 等の制御構文は空白を残す（VS標準）

### 項目4: タブによる位置合わせを解除

変数宣言・メンバー変数・#define 等の複数タブによる桁揃えを、タブ1つに変更する。

**変更前:**
```cpp
BYTE				m_byLevel;						/* 明度 */
BYTE				m_byFadeState;					/* フェード状態 */
DWORD				m_dwLastFade;					/* 前回のフェード処理 */
```

**変更後:**
```cpp
BYTE	m_byLevel;		/* 明度 */
BYTE	m_byFadeState;	/* フェード状態 */
DWORD	m_dwLastFade;	/* 前回のフェード処理 */
```

- 型名と変数名の間: タブ1つ
- 行末コメントの位置合わせ: 不要（自然な位置でOK）

### 項目5: セクションコメントの簡略化

ファイル内のセクション区切りコメントを簡略化する。

**変更前:**
```cpp
/* ========================================================================= */
/* クラス宣言																 */
/* ========================================================================= */
```

**変更後:**
```cpp
// クラス宣言
```

- 定数の定義、構造体の定義、関数プロトタイプ等のセクション見出しも同様

### 項目6: `/* */` コメントを `//` に統一

コード中の `/* ... */` コメントを `//` 形式に統一する。

**変更前:**
```cpp
/* SDLウィンドウのHWNDを受け取ってゲーム側を初期化する */
/* OnCreate / OnInitEnd 相当の処理をここで行う         */
/* 失敗時はFALSEを返す                                 */
virtual BOOL OnSDLInit(SDL_Window *pWindow) = 0;
```

**変更後:**
```cpp
// SDLウィンドウのHWNDを受け取ってゲーム側を初期化する
// OnCreate / OnInitEnd 相当の処理をここで行う
// 失敗時はFALSEを返す
virtual BOOL OnSDLInit(SDL_Window *pWindow) = 0;
```

- 行末コメント `/* xxx */` → `// xxx` も対象
- 複数行にまたがる `/* ... */` は各行 `//` に変換

## 進め方

1. 小規模ディレクトリ（Platform → Lib → MainFrame）から着手し、パターンを確立
2. 各ディレクトリ完了時にコンパイル確認（関連ファイルのみ）
3. 完了ごとにこのドキュメントの「状態」を更新
4. 問題が見つかった場合はこのドキュメントに記録

## 注意事項

- `Lib/AflMusic/` は外部ライブラリ由来の可能性があるため、初回は様子を見て判断
- `if (`, `for (`, `while (` 等の制御構文の空白は削除しない
- `typedef class` / `typedef struct` の書き換えは今回のスコープ外（構造変更を伴うため別タスクとする）
- `goto` パターン、`SAFE_DELETE` マクロ等も今回はスコープ外
