# Web 管理画面 機能リファレンス

> 調査日: 2026-06-19  
> 対象: `SboSvr/webroot/js/views/` 配下の全 28 ファイル  
> コード裏取り済み。計画文書（web-admin-map-plan.md 等）は参考程度とした。

---

## 全体像

Web 管理画面は、SboSvr が提供する REST API（`/api/...`）をブラウザから呼び出すシングルページアプリケーションである。ハッシュルーティング（`#map-info` 等）で画面を切り替え、各 view ファイルが 1 画面に対応する。

機能の中心は **マップ編集**（タイル描画・パーツ/オブジェクト配置・イベント設定）と **マスタデータ管理**（アイテム・武器・スキル・エフェクト・モーション）の 2 軸である。キャラクター・アカウント管理と運用補助ツール（ダッシュボード・監査ログ）も揃っており、日常的な運用はほぼ GUI で完結する設計になっている。ゲーム画面（iframe）とリアルタイムで連携し、地図クリックでイベント座標やオブジェクト配置座標を受け取ることができる。ドロップアイテムの「どの敵が何をドロップするか」のマスタ設定画面は現時点では存在せず、アイテムの個別インスタンス（落下/所持状態）を直接編集する画面のみがある。

## Phase A に直結する重要ポイント

### 1. ワープ/マップ移動イベント（ポータル）

**結論: GUIで完結する。**

`map-events.js` でマップイベント種別「2: マップ間移動 (MAPMOVE)」を選択すれば、ポータルを作成・編集できる。操作手順:

1. 「マップイベント編集」画面を開き、対象マップを選択する。
2. 「新規追加」ボタンを押し、フォームを開く。
3. 種別を「マップ間移動 (MAPMOVE)」に設定する。
4. 固有フィールドとして「移動先マップ ID」・「移動先 X」・「移動先 Y」・「向き」を入力する。
5. 共通フィールド: 当たり判定種別（MAPPOS/CHARPOS/AREA/MAPPOS2）・判定向き・座標1(X/Y)・座標2(X/Y)・効果音 ID を設定する。

ゲーム iframe 連携でクリック座標を取得することも可能（`window._mapEventsState` 経由で座標反映）。

### 2. 敵スポーナー/NPC 配置

**結論: GUIで完結する。** スポーナーの設定はキャラ情報画面（char-edit.js）の「NPC発生」タブで編集・保存できる（実機確認済み）。

- **スポーナーのパラメータ**は `char-edit.js`「NPC発生」タブで設定: 発生周期(putCycle)・発生させる移動種別(putMoveType)・同時発生数(maxPutCount)・発生確率(putAverage)・発生範囲X/Y(putAreaX/Y)。`buildNpcSpawnTab()` 参照。サーバ側の `CHARMOVETYPE_PUTNPC` / `m_ptPutArea` 等に対応。
- つまり「親キャラ＝スポーナー」をキャラ情報画面で作り、発生範囲・周期・湧き数・発生させる敵の移動種別を指定すれば、範囲湧きが成立する。
- `npc-add.js` では NPC を 1 体ずつ指定座標に個別配置することもできる（固定NPC用）。

### 3. ドロップアイテムの設定

**結論: GUIで完結する。** キャラ情報画面（char-edit.js）でドロップ率・落とす品ともに設定・保存できる（実機確認済み）。

- **ドロップ率**(dropItemAverage) は `char-edit.js`「移動」タブで設定（`buildMovementTab()`、サーバ側 `m_nDropItemAverage` に対応）。
- **落とす品（ドロップ元）** は `char-edit.js`「所持アイテム」タブで NPC の所持品として登録する。
- 補助: `item-types.js`（種別マスタ CRUD）、`item-list.js`（アイテムインスタンスを地面座標へ落下配置/キャラのバックパックへ投入）。

## 不明だった点・要確認

- **`map-paint.js` の保存エンドポイント**: タイル描画の保存先 API パスがコードのオプション引数 (`onSave`) 経由で呼び出し元に委ねられており、単体では不明。`map-parts-place.js` 側から組み込まれるが最終的な API パスは要確認。
- **マップ影カタログ `map-shadow-edit.js`**: コメントに API パスの記載なし。`/api/maps/shadows` 相当のエンドポイントがあると推定されるが未確認。
- **モーション編集 `motion-edit.js`**: API パスがコメントに記載なし（`/api/motions` 相当と推定）。
- **NPC のリポップ/スポーナー設定**: `npc-add.js` では固定座標への配置のみ確認できた。リポップ間隔等がサーバー側テーブルに存在するか、または NPC 種別の `moveType` だけで制御されているかは C++ 側コードの確認が必要。
- **ドロップテーブル**: アイテム種別マスタ・NPC マスタのいずれにも「ドロップ率」フィールドが管理画面コードに見当たらない。SQLite テーブルに `drop_item` 系のテーブルが存在するか要確認。
- **`audit-reports.js` / `failback-guide.js`**: どちらも Wave 3 向けプレースホルダーで未実装。
- **`character-list.js` / `character-account.js`** の一部 API: `char-edit.js` と機能が一部重複しており、実際の使い分けルールを把握したい場合は整理が必要。
