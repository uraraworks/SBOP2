# 自由移動システム移行計画

## 1. 背景と目的

現在のシステムはタイル半分（16px）単位の移動・スクロールを採用している。
これを Core Keeper 等の現代的な 2D ゲームで主流の方式——当たり判定はタイル単位のまま、キャラの動きは数ピクセル単位の自由移動、カメラはキャラ位置に追随——へ移行する。

### 現状の仕組み（変更前）

| 要素 | 現状 |
|------|------|
| タイルサイズ | 32 × 32 px |
| 内部座標単位 | 0.5 タイル = 16 px（`m_nMapX`, `m_nViewX` 等が 2 倍値） |
| キャラ移動単位 | 16 px（1 ステップ = タイル半分） |
| スクロール単位 | 16 px（`SCROLLSIZE = 16`） |
| スクロール契機 | キャラが画面端付近に達したとき段階的に 16px 刻みでスクロール |
| 当たり判定 | 内部座標（2 倍値）→ `÷2` → タイル座標 で判定 |

### 目標の仕組み（変更後）

| 要素 | 目標 |
|------|------|
| タイルサイズ | 32 × 32 px（変更なし） |
| 内部座標単位 | 1 px 単位（整数ピクセル座標で管理） |
| キャラ移動単位 | N px/フレーム（速度定数で調整可能、例: 2〜4 px） |
| スクロール | キャラ中心を画面中央に追随するカメラ方式（サブピクセル対応） |
| 当たり判定 | キャラのピクセル座標をタイル座標へ変換して既存タイル判定を流用 |

## 3. 設計方針

### 3.1 座標系の再定義

変更後の座標系を 1 px 単位で統一する。

```
ワールド座標(px単位)
    ÷ MAPPARTSSIZE(32)
タイル座標(当たり判定・マップデータ参照)

ワールド座標(px単位)
    - カメラ座標(px単位)
スクリーン座標(px単位)
```

**現状の `m_nMapX` が 2 倍値になっている理由**は、0.5 タイル精度を整数で表現するためだった。
移行後は px 単位なので 2 倍値不要。`m_nMapX` はそのままピクセル値として扱う。

### 3.2 カメラ（スクロール）方式

```
カメラ中心 = キャラのワールド座標(px)
カメラ左上 = カメラ中心 - (画面サイズ / 2)
描画オフセット = タイル座標 × MAPPARTSSIZE - カメラ左上
```

- マップ端ではカメラをクランプ（端より外を映さない）
- 将来的にカメラの「遅延追随（smooth follow）」も実装可能な設計にする

### 3.3 当たり判定の流用

現在の `IsMove(int tileX, int tileY, int nDirection)` はタイル単位の判定で正しく機能している。
移行後はキャラのピクセル座標をタイル座標に変換して渡すだけで流用できる。

```cpp
// 移行後のピクセル座標→タイル座標変換
int tileX = pixelX / MAPPARTSSIZE;
int tileY = pixelY / MAPPARTSSIZE;
// 既存の IsMove をそのまま使用
IsMove(tileX, tileY, direction);
```

移動先の当たり判定は「キャラの足元矩形の各頂点がすべて通行可能か」を確認する方式とする。

### 3.4 経路探索（RouteSearch）の対応

RouteSearch はタイル単位で動作しているため、入出力の変換のみ対応する。

```cpp
// 現状（0.5タイル単位を /2 してタイル座標へ変換）
SetStartPos(m_nMapX, m_nMapY);   // 内部で /2 処理あり

// 移行後（ピクセル座標を /MAPPARTSSIZE してタイル座標へ変換）
SetStartPosPx(pixelX, pixelY);   // 内部で /MAPPARTSSIZE 処理
```

RouteSearch の探索結果はタイル座標で返ってくるため、移動目標をタイル中心のピクセル座標へ変換して使用する。

### 3.5 移動速度

現状の「1 ステップ = 16px / フレーム」を廃止し、定数で速度を管理する。

```cpp
// GlobalDefine.h または SboCli_priv.h に追加
#define CHAR_MOVE_SPEED   2    // キャラ移動速度 (px/フレーム)
```

速度はアイテムやスキルによってバフ/デバフを掛けられるよう `InfoCharBase` に係数フィールドを持たせる。

## 4. 影響範囲と変更量の見積もり

### 4.1 変更レベル分類

| レベル | 内容 |
|--------|------|
| **High**（大規模改修） | 座標系の根幹変更・全描画処理 |
| **Medium**（中規模） | 入力処理・経路探索インターフェース |
| **Low**（小規模） | 定数変更・変換式の追加 |

### 4.2 ファイル別変更規模

| ファイル | 規模 | 変更内容 |
|---------|------|---------|
| `Common/Info/InfoCharBase.h/.cpp` | High | `m_nMapX/Y` をピクセル単位に変更、座標取得関数全面修正 |
| `SboCli/src/Layer/LayerMap.cpp/.h` | High | `m_nViewX/Y` をピクセル単位に変更、カメラ追随ロジック新設、描画オフセット計算全面修正 |
| `SboCli/src/MgrDraw.cpp` | High | 全キャラ・オブジェクト描画の座標計算を新座標系対応 |
| `SboCli/src/StateProc/StateProcMAP.cpp` | High | 入力→移動処理をピクセル移動に変更、スクロール処理廃止 |
| `Common/Info/InfoMapBase.h` | Medium | `IsMove` 等の入力座標系の確認と調整 |
| `Common/RouteSearch.cpp/.h` | Medium | 入出力の変換レイヤー追加 |
| `SboCli/src/Info/InfoCharCli.cpp/.h` | Medium | `m_ptMove` 補間処理の見直し |
| `SboCli/src/SboCli_priv.h` | Low | `SCROLLSIZE` 廃止、`CHAR_MOVE_SPEED` 追加 |
| `Common/GlobalDefine.h` | Low | 移動関連定数の整理 |
| `SboSvr` 側（サーバー） | Medium | サーバー上の `m_nMapX/Y` も座標系を合わせる必要あり |

> **サーバー側への影響について**
> サーバー（`SboSvr`）もキャラ座標を `m_nMapX/Y`（0.5 タイル単位）で管理している。
> クライアントだけ変えるとサーバーとの整合性が崩れるため、**サーバー側の座標系も同時に移行**する必要がある。
> ネットワークパケットの座標フィールドのビット幅・スケールも見直す。

## 5. 実装フェーズ

### Phase 1: 座標系の定義整理と変換関数の追加（準備）

**目的:** 新旧座標系が共存できる橋渡し関数を用意し、段階移行の基盤を作る。

**作業内容:**
- `MAPPARTSSIZE`（32）を唯一の変換係数として確立
- 変換ユーティリティ関数を追加
  ```cpp
  // タイル座標 ↔ ピクセル座標
  inline int TileToPixel(int tile) { return tile * MAPPARTSSIZE; }
  inline int PixelToTile(int px)   { return px / MAPPARTSSIZE; }
  inline int PixelToTileCenter(int px) { return PixelToTile(px) * MAPPARTSSIZE + MAPPARTSSIZE / 2; }
  ```
- 旧 `SCROLLSIZE`・2 倍座標系の使用箇所を全列挙（grep）

**完了基準:** 変換関数が定義され、既存コードが変更なくビルド通る。

---

### Phase 2: `InfoCharBase` の座標系移行（コア）

**目的:** キャラクター座標をピクセル単位へ変更し、当たり判定・座標取得関数を対応させる。

**作業内容:**
- `m_nMapX`, `m_nMapY` の単位を 0.5 タイル → px に変更
- `GetPosRect`, `GetMapPosRect`, `SetPos`, `IsHitCharPos` 等の座標計算を修正
- `RenewBlockMapArea` をタイル座標で動作するよう修正（内部で `PixelToTile` 変換）
- サーバー側 `InfoCharSvr` も同様に対応

**完了基準:** キャラの座標取得・設定・当たり判定がすべて px 座標で正しく動作する。

---

### Phase 3: `LayerMap` のカメラ追随スクロールへ移行（描画）

**目的:** 段階スクロールを廃止し、カメラ追随方式の描画オフセット計算に変更する。

**作業内容:**
- `m_nViewX`, `m_nViewY` をサブピクセル対応の px 値に変更
- `Scroll`, `IsScrollPos`, `IsScrollArea`, `SetCenterPos` を廃止または大幅改修
- カメラ位置計算関数を新設
  ```cpp
  void UpdateCamera(int charPxX, int charPxY);
  // カメラをキャラ中心に追随、マップ端でクランプ
  ```
- 描画ループの各オブジェクト座標計算を新方式に対応
- `MgrDraw` のキャラ・オブジェクト描画の座標計算も対応

**完了基準:** キャラが移動するとカメラが滑らかに追随し、マップ端でクランプされる。

---

### Phase 4: `StateProcMAP` の移動処理変更（入力・ロジック）

**目的:** 入力→移動処理をピクセル単位の自由移動へ切り替える。

**作業内容:**
- 移動処理を「N px/フレームの方向移動」に変更
- 移動前に当たり判定（各頂点チェック）を行い、壁押し出し処理（軸分離）を追加
- 経路探索結果の目標座標をタイル中心 px 座標へ変換
- `MOVEWAIT` 等の古い移動周期定数を廃止

**完了基準:** 壁に当たらず、自由方向に数 px 単位で移動できる。

---

### Phase 5: 経路探索・NPC 移動の対応（AI）

**目的:** RouteSearch の結果を新座標系で正しく消費できるようにする。

**作業内容:**
- RouteSearch への入力・出力を px ↔ タイルで変換するラッパーを追加
- NPC の移動ロジックも Phase 4 と同様にピクセル移動へ対応
- サーバー側の NPC 移動・座標同期パケットを新座標系へ対応

**完了基準:** NPC が経路探索の結果に従い自由移動できる。

---

### Phase 6: ネットワーク座標同期の更新（通信）

**目的:** クライアント・サーバー間の座標パケットを新座標系へ移行する。

**作業内容:**
- 座標を含むすべてのパケット定義（`PacketCHAR_*` 等）を確認
- 座標フィールドのビット幅・スケールを px 単位で統一
- 既存の 0.5 タイル単位の座標値を px 値へ変換（マイグレーション）
- セーブデータの座標もスケール変換（既存セーブとの互換性を確認）

**完了基準:** クライアントとサーバーが同じ px 座標で同期できる。

---

### Phase 7: 統合テストと調整

**目的:** 全体を通して動作確認し、移動感覚を調整する。

**作業内容:**
- 移動速度（`CHAR_MOVE_SPEED`）の数値調整
- カメラ追随のクランプ動作の確認
- マップ境界付近での描画崩れ確認
- NPC との衝突・すり抜けが起きないか確認
- マップイベント・会話等のトリガー判定が新座標系で正しく動くか確認

---

## 6. 非目標（この計画で実施しない）

- カメラの遅延追随（イージング）実装（Phase 7 後の追加機能として検討可能）
- タイルサイズの変更（32px のまま維持）
- アニメーション周りの大幅変更（スプライト差し替え等）
- 斜め当たり判定の完全物理シミュレーション化

---

## 7. リスクと対策

| リスク | 対策 |
|--------|------|
| サーバー・クライアント座標不整合 | Phase 2〜3 完了後にプロトタイプサーバーで早期確認 |
| 既存セーブデータの座標ズレ | 読み込み時に旧→新スケール変換処理を追加（`× MAPPARTSSIZE / 2`） |
| マップイベントのトリガー判定ズレ | イベント座標もタイル単位で保持し、ピクセル変換で対応 |
| NPC が壁にめり込む・すり抜ける | 軸分離の衝突解決（AABB sliding）を適切に実装 |
| 作業量が多く途中で混在状態が長くなる | Phase 単位でブランチを切り、各 Phase を単体で動作確認 |

---

## 8. まとめ

この移行は **影響範囲が広く大規模**だが、以下の理由から実現可能と判断できる。

- 当たり判定（`IsMove`）はタイル単位のまま流用でき、完全な作り直しは不要
- 座標変換は `÷ MAPPARTSSIZE` の単純な変換で旧タイル判定を利用できる
- フェーズを分けることで各段階ごとに動作確認しながら進められる

現行の「0.5 タイル = 16px の 2 倍値」という複雑な座標系を廃止して px 単位に統一することで、
長期的なコードの保守性も向上する。
